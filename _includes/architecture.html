Architecture

Overview

Olympus has a modular architecture, with two primary components: Pythia (compute) and Phalanx (storage). Each component can be deployed, scaled and used independently. Compute resources can be provisioned using container and virtual machine images. While storage resouces can be accessed through an unified object, block and file interface.



Phalanx

Phalanx is the storage engine component of Olympus. It offers storage services to clients through an unified object, block and file interface. Virtual machines, containers, applications and users access data either locally or through a shared network. Phalanx is designed and optimized for modern software-defined environments and next generation storage devices. Phalanx's novel row & column architecture, device-aware design and log structured data layout minimize IO contention and latency while preventing hot write areas. Phalanx can also be easily containerized and deployed using existing orchestration frameworks (e.g. Kubernetes).

Phalanx IO stack



Phalanx's request management layer intelligently prioritizes incoming requests, enabling clients to achieve high throughput and low latency IO. A separate metadata store increases the scalability, efficiency and robustness of the system. In addition, the cloud connect technology enables clients to seamlessly utilize resources from multiple cloud providers.

Phalanx device management



Phalanx implements a novel row/column architecture for device management. It is designed for scalability and performance - adding a device scales both performance and storage capacity of the system.



Log structured data layout



Together with device-aware and log structured data layout, Phalanx minimizes Read/Write IO contention and evenly distributes write workload across capacity preventing hot write areas. Phalanx is also host SMR capable (implements strict sequential writing) and supports variable capacity devices.
