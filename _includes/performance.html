Every application workload is different and every business requirement unique - no single benchmark nor trace can capture the whole user experience. However, it is useful to profile IO performance and usage patterns before committing to a storage solution or configuration. Below is a simple trace comparison between Olympus and an ext4 file system to show what makes Olympus fast and efficient.

Above shows what happens when we write twenty 100MB files sequentially using 64KB record size. With ext4, the storage device executed approximately 287 seeks (12.59 avg seek/sec * 22.87 sec) to complete the write. In contrast, Olympus wasted no device cycles on seeks and translated that efficency into a better overall throughput (MB/s) and latency (IOPS) for the user.
